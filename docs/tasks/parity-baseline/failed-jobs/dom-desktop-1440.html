What does this mean?

A failed job is a background task — video index or cloud upload — that retried multiple times and gave up. Each card shows what failed and a recommendation for what to do.

Retry resets the row so the worker picks it up again. Delete removes the queue row only — it never deletes the underlying video file from disk. Producers (file watcher, boot scan) may legitimately re-enqueue the same file later if it still exists.

Clip value badge: Event clip = irreplaceable. Rolling buffer = rotated hourly. Map / trip data = re-derivable.
