Storage

Allocate how much of the SD card each partition presents to Tesla, and tune the auto-cleanup that keeps the TeslaCam partition from filling up. The gadget exposes a single USB drive with two exFAT partitions (TeslaCam and Media). You can grow a partition into the card's free space — the maximum is capped so the SD can never physically overfill and crash the device. Changing a partition's size briefly disconnects the USB gadget (~30–60 s) while Tesla re-enumerates.

Partition sizes

Each partition can grow into the card's free space. The maximum below is computed live as SD total − measured OS usage − the other partition − safety buffer, so the two partitions plus the OS can never physically overfill the card. You can't shrink a partition below the data it currently stores.

Shrinking a partition below its current usage is refused. Size changes regenerate the synthesized exFAT geometry over the same Linux directory tree — your files are not touched, and Tesla will see the new size on the next reconnect without any reformat.

Auto-cleanup (TeslaCam)

The worker continuously sweeps the TeslaCam partition when free space dips below the target. Deletion priority: Tier A — RecentClips with no GPS or SEI data, then Tier B — RecentClips with GPS/SEI, then Tier C — SavedClips and SentryClips. SentryClips are only touched when an age threshold is set.

Preserve clips with GPS/SEI

When on, GPS/SEI clips move to Tier B and are deleted only after plain Tier A clips are exhausted.
